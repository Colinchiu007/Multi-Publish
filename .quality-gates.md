### 规则 1：路径变更必须用 path-utils
任何涉及 `__dirname` + `..` 的路径计算，**必须**使用 `path-utils.js` 提供的函数：
- `getProjectRoot()`
- `getComposerDir()`
- `getPythonBackendDir()`
- `getConfigPath(filename)`

**例外**：仅当文件在 `packages/` 内部且路径层级正确时，可直接使用 `path.join(__dirname, ...)`。

### 规则 2：新增服务文件必须注册到 path-utils
如果在 `electron/services/` 下新增需要访问项目根目录/配置文件的服务，**必须**：
1. 在 `path-utils.js` 中添加对应的 getter 函数
2. 服务文件通过 `require('./path-utils')` 调用

### 规则 3：远程部署兼容性
任何新增的文件路径引用，**必须**支持通过环境变量覆盖：
```js
// 正确示例
const root = process.env.MULTI_PUBLISH_ROOT || getDefaultRoot();

// 错误示例（硬编码）
const root = '/home/user/Multi-Publish';
```

### 规则 4：异步测试必须真实等待
异步测试必须使用 `node:test`、Vitest 或显式等待 Promise 的统一测试工具。禁止用
`try { fn() }` 一类同步包装器执行 `async` 测试；提交前必须在不含工作区未提交改动的
干净提交快照中运行模块全量测试，不能用脏工作区的通过结果代替。

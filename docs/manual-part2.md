### 23.1 功能概述

> ![云端发布页面](img_v3_02139_3c7e1778-b406-4e3b-9ccd-4b77ceb2324g)
> *图 41：云端发布 — 将发布任务提交到 ECS 云服务器执行*

云端发布允许将发布任务提交到 ECS 云服务器执行，无需本地持续运行：

| 特性 | 说明 |
|------|------|
| **提交发布** | 本地编辑内容 → 提交到云端 → 云服务器执行 RPA |
| **任务列表** | 查看所有已提交的云端任务 |
| **进度轮询** | 前端定时轮询任务状态 |
| **即关即走** | 提交后关闭 App 不影响任务执行 |

### 23.2 支持的平台

目前支持云端发布的平台：

- **B站**（bilibili_publisher.py — Cookie-based 直连发布）
- **抖音**（douyin_publisher.py — Cookie-based 直连发布）

### 23.3 使用方式

1. 左侧导航 → **「☁️ 云端发布」**
2. 选择发布模式：
   - **`mode: "rpa"`**：本地 RPA 执行（默认）
   - **`mode: "cloud"`**：提交到云端执行
3. 编辑内容 → 选择平台 → 点击发布
4. 任务提交到云端 → 可在任务列表查看进度
5. 关闭 App 后重新打开 → 任务继续执行不受影响

### 26.3 插件开发规范

```javascript
// 完整的插件示例
module.exports = {
  // 必填：平台唯一标识
  platform: 'my_platform',

  // 必填：界面显示名称
  displayName: '我的平台',

  // RPA 发布模式（选填，至少实现一种发布方式）
  async publish(page, article) {
    // page: RpaViewManager 的 BrowserWindow 实例
    // article: { title, content, images[], coverUrl }
    // 编写 DOM 操作逻辑...
    return { success: true, url: 'https://...' };
  },

  // API 发布模式（选填）
  async publishViaApi(article) {
    // 通过 HTTP API 发布
    const response = await fetch('https://api.example.com/publish', {
      method: 'POST',
      body: JSON.stringify(article)
    });
    return response.json();
  },

  // 登录验证（选填）
  async validate(cookies) {
    // 返回 true/false 表示 Cookie 是否有效
    return true;
  }
};
```

> 详细开发指南见 `docs/plugin-development-guide.md`。

## 二十七、API 开放平台

> 功能自 v1.7.0 起可用

### 27.1 概述

> ![API 管理页面](img_v3_02139_b57fe2cf-0958-43aa-905e-95ac243d2f9g)
> *图 46：API 管理 — 创建和管理 API Key*

API 开放平台允许开发者通过 REST API 调用 Multi-Publish 的发布能力，适用于 CI/CD 集成、自动化工作流等场景。

### 27.2 API Key 管理

**创建 Key：**

1. 设置 → **「API 管理」**
2. 点击 **「创建 API Key」**
3. 设置权限范围（scope）：
   - `*` 通配符：全部权限
   - `publish:read`：只读发布任务
   - `publish:write`：提交发布任务
   - `admin`：管理权限（创建/吊销 Key）
4. 生成的 Key 格式：`mp_` + 48 位十六进制字符
5. **复制并保存 Key**（关闭后不再显示原文）

**管理 Key：**

| 操作 | 端点 | 说明 |
|------|------|------|
| 创建 | `POST /api/v1/keys` | 新建 Key（需 admin scope） |
| 列表 | `GET /api/v1/keys` | 查看所有 Key（不暴露原文） |
| 吊销 | `DELETE /api/v1/keys/:key` | 吊销指定 Key |

### 27.3 API 端点

| 方法 | 端点 | 说明 |
|:----:|------|------|
| GET | `/api/v1/platforms` | 获取支持平台列表 |
| POST | `/api/v1/publish` | 提交发布任务 |
| GET | `/api/v1/jobs/:id` | 查询任务状态 |
| POST | `/api/v1/keys` | 创建 API Key（admin） |
| GET | `/api/v1/keys` | 列出 API Key（admin） |
| DELETE | `/api/v1/keys/:key` | 吊销 API Key（admin） |

### 27.4 SDK 使用

```bash
npm install @multi-publish/api-publish-engine
```

```javascript
const { PublishApiClient } = require('@multi-publish/api-publish-engine');

const client = new PublishApiClient({
  apiKey: 'mp_abc123...',
  baseUrl: 'http://localhost:16521' // 默认
});

// 获取平台列表
const platforms = await client.getPlatforms();

// 提交发布任务
const job = await client.publish({
  platform: 'weixin',        // 平台标识
  title: '文章标题',
  content: '# Markdown 正文...',
  coverUrl: 'https://...',   // 封面图 URL（可选）
  scheduledAt: '2026-07-10T10:00:00Z' // 定时发布（可选）
});

// 查询任务状态
const status = await client.getJobStatus(job.id);
console.log(status); // { id, status: 'running', progress: 60 }
```

### 27.5 Webhook

> 功能自 v1.6.1 起可用

配置 Webhook 接收发布状态变更通知：

| 设置项 | 说明 |
|--------|------|
| **Webhook URL** | 接收通知的 HTTP 端点 |
| **触发事件** | `publish.success` / `publish.failed` / `publish.started` |
| **签名验证** | 请求带 `X-Signature` 头部验证来源 |

**Webhook Payload 格式：**

```json

  "event": "publish.success",

    "title": "文章标题",
    "url": "https://mp.weixin.qq.com/s/..."
  }
}
```

| **健康检测** | 自动检测代理可用性 |
| **失败切换** | 代理不可用时自动切换到下一个 |

### 29.9 日志系统

| 日志类型 | 位置 | 用途 |
|---------|------|------|

| **RPA 日志** | `logs/rpa.log` | RPA 引擎操作日志 |

## 三十、常见问题

### 21.1 安装与启动

**Q: 安装包下载很慢怎么办？**

A: 推荐以下方式：
- 使用 **Gitee 国内镜像**：https://gitee.com/ColinChiu007/Multi-Publish/releases（5MB/s）
- 使用 `gh-proxy.com` 加速：`https://gh-proxy.com/https://github.com/Colinchiu007/Multi-Publish/releases/download/v1.1.0/Multi-Publish.Setup.1.1.0.exe`

**Q: 首次启动很慢，卡在"检测依赖"？**

A: 首次启动需要下载内置浏览器组件（~170MB），请耐心等待 1-2 分钟。如果超过 5 分钟无响应：
1. 检查网络连接
2. 检查是否被防火墙拦截
3. 重启 App 重试
4. 首次运行后，后续启动秒开

**Q: macOS 提示"无法验证开发者"？**

A: 由于未进行 Apple 开发者签名，首次打开需要：
1. 右键点击 App → 选择「打开」
2. 在弹出的对话框中选择「打开」
3. 之后即可正常打开

**Q: Linux 下 AppImage 无法运行？**

A: 执行 `chmod +x Multi-Publish-*.AppImage` 赋予执行权限后运行。如果提示 FUSE 错误，添加 `--no-sandbox` 参数。

### 21.2 账号相关

**Q: 添加账号时浏览器窗口不弹出？**

A: 检查：
1. 是否被防火墙拦截
2. 操作系统弹窗权限是否开启
3. 是否有其他 Electron 应用占用端口

**Q: 登录状态经常过期怎么办？**

A: 各平台登录有效期不同（微信扫码通常 7-30 天，账号密码可能 1-3 天）。建议：
- 定期检查账号状态（状态指示灯 🟢=正常 🟡=即将过期 🔴=已过期）
- 收到重新登录提示时及时处理
- 微信公众号使用扫码登录有效期更长

**Q: Cookie 安全吗？会被上传到服务器吗？**

A: 绝对安全：
- 所有 Cookie 使用 **AES-256-GCM** 加密后存储在本地文件
- **不会上传到任何远程服务器**
- 每个账号使用 `session.fromPartition()` 独立 Cookie 分区
- 加密密钥从本地机器特征派生

**Q: 最多能添加多少个账号？**

A: 没有数量限制。但建议单平台不超过 10 个账号，过多的账号同时发布可能被平台识别。

### 21.3 发布相关

**Q: 发布失败怎么办？**

A: 按以下步骤排查：

| 步骤 | 检查项 | 解决方法 |
|:----:|--------|---------|
| ① | 账号登录状态 | 重新登录过期账号 |
| ② | 查看失败原因 | 任务详情中显示具体错误信息 |
| ③ | 内容格式 | 检查是否有不兼容的格式 |
| ④ | 敏感词 | 运行敏感词检测 |
| ⑤ | 重试 | 修复后点击「重新发布」 |

**常见失败原因：**

| 原因 | 处理方式 |
|------|---------|
| Cookie 过期 | 重新登录 |
| 内容违规（敏感词） | 修改内容后重试 |
| 平台限流 | 等待 5 分钟后重试 |
| 网络异常 | 检查网络连接 |
| 平台 UI 变更 | 更新到最新版本 |

**Q: 视频上传支持哪些格式和大小？**

A: 支持 `mp4`、`mov`、`avi` 格式，建议不超过 500MB。各平台对视频时长和分辨率有不同限制，发布前请参考平台官方要求。

**Q: 定时任务重启后还在吗？**

A: 在。定时任务持久化到 SQLite 数据库，App 重启后自动恢复。

**Q: 为什么提示"发布间隔太短"？**

A: 同一账号两次发布之间至少需要间隔 **5 分钟**，这是为了防止被平台限流封号。等待 5 分钟后自动重试。

**Q: 批量发布时，一个平台失败会影响其他平台吗？**

A: **不会**。每个平台的发布任务互相独立，一个平台失败不影响其他平台的发布。

### 21.4 数据与存储

**Q: 数据存储在哪里？**

A: 所有数据存储在本地应用数据目录：

| 系统 | 默认路径 |
|:----:|---------|
| Windows | `%APPDATA%/Multi-Publish/` |
| macOS | `~/Library/Application Support/Multi-Publish/` |
| Linux | `~/.config/Multi-Publish/` |

**Q: 如何备份数据？**

A: 备份应用数据目录下的 `database.sqlite` 文件和 `config/` 目录即可。

**Q: 更换电脑如何迁移数据？**

A: 将旧电脑上的应用数据目录整个复制到新电脑的对应位置，覆盖后重启 App。

### 21.5 其他

**Q: 能否同时在多台电脑使用？**

A: 免费版限 1 台，Pro 版限 3 台。

**Q: 如何联系技术支持？**

A: 通过以下渠道：
- **GitHub Issues**：https://github.com/Colinchiu007/Multi-Publish/issues
- **Pro 用户**：享有优先级支持（48 小时内回复）

**Q: 如何提交功能建议？**

A: 在 GitHub Issues 提交 Feature Request，或通过设置页的「反馈」入口提交。

**Q: 是否会收集用户数据？**

A: 仅收集使用统计数据（如发布次数、功能使用率），用于改善产品体验。**不会收集**内容数据、账号密码、Cookie 等敏感信息。

# 蚁小二功能复用方案

> 基于截图分析和代码结构分析制定
> 版本: 1.0
> 日期: 2026-07-16

---

## 1. 复用目标

### 1.1 总体目标
在 Multi-Publish 项目中复刻蚁小二的全部功能，实现：
- 除菜单布局外，所有界面和功能与蚁小二一致
- 复用蚁小二的交互设计和用户体验
- 保留 Multi-Publish 的技术优势（RPA 引擎）

### 1.2 具体指标
| 指标 | 目标 |
|------|------|
| 功能覆盖率 | ≥ 90% |
| UI 相似度 | ≥ 80% |
| 代码复用率 | ≥ 50% |
| 开发周期 | 6 个月 |

---

## 2. 复用策略分级

### 2.1 A 级：直接复用

可直接使用的 Multi-Publish 现有模块：

| 模块 | 路径 | 功能 | 复用方式 |
|------|------|------|---------|
| credential-store.js | electron/services/ | AES-256-GCM 加密存储 | 直接使用 |
| account-state-restorer.js | electron/services/ | 登录状态持久化 | 直接使用 |
| publish-monitor.js | electron/services/ | 发布状态监控 | 直接使用 |
| api-platform-adapter.js | electron/services/ | API 发布适配器 | 直接使用 |
| webview-manager.js | electron/services/ | 分屏监控 | 直接使用 |
| rpa-engine | packages/ | RPA 发布引擎 | 直接使用 |

### 2.2 B 级：参考实现

需要参考蚁小二设计，但需要重写：

| 模块 | 蚁小二参考 | Multi-Publish 现状 | 实现方式 |
|------|-----------|-------------------|---------|
| 登录 UI | 截图 + PRD | 部分实现 | 参考蚁小二设计重写 |
| 账号管理 UI | 截图 + PRD | 基础实现 | 参考蚁小二设计重写 |
| 发布编辑器 | 截图 + PRD | RPA 模式 | 新开发富文本编辑器 |
| 草稿箱 | 截图 + PRD | 无 | 新开发 |
| 定时发布 | 截图 + PRD | 部分实现 | 完善定时功能 |
| 评论管理 | 截图 + PRD | 基础实现 | 增强功能 |

### 2.3 C 级：从零实现

蚁小二有但 Multi-Publish 需要全新开发：

| 模块 | 说明 | 优先级 |
|------|------|--------|
| 数据统计 | 发布数据、阅读量、互动数据 | P2 |
| 团队管理 | 多用户协作、权限管理 | P2 |
| AI 助手 | AI 写作、数据分析 | P1 |
| 素材库云同步 | 多设备同步 | P2 |
| 高级工具箱 | 图片处理、视频剪辑 | P2 |

## 4. 技术方案

### 4.1 前端架构

```
src/
├── views/                  # 页面组件
│   ├── login/              # 登录页
│   ├── home/               # 首页
│   ├── publish/            # 发布页
│   ├── accounts/           # 账号管理
│   ├── comments/           # 评论管理
│   ├── material/           # 素材库
│   ├── data/               # 数据统计
│   ├── team/               # 团队管理
│   └── settings/           # 设置
├── components/             # 通用组件
│   ├── ui/                 # 基础 UI
│   ├── editor/             # 编辑器
│   └── charts/             # 图表
├── stores/                 # Pinia 状态
└── composables/            # 组合式函数
```

### 4.2 后端架构

```
electron/
├── main.js                 # 主进程入口
├── services/               # 服务模块
│   ├── auth/               # 认证服务
│   ├── account/            # 账号服务
│   ├── publish/            # 发布服务
│   ├── comment/            # 评论服务
│   └── storage/            # 存储服务
└── ipc/                    # IPC 处理
```

### 4.3 技术选型

| 模块 | 技术 | 说明 |
|------|------|------|
| 图标 | lucide-vue-next | 蚁小二同款图标 |
| 图表 | ECharts | 数据可视化 |
| 拖拽 | @vueuse/core | 拖拽功能 |
| 日期 | dayjs | 日期处理 |
| 加密 | crypto-js | AES 加密 |

## 5. 质量保障

### 5.1 测试策略
- 单元测试：覆盖率 ≥ 60%
- 集成测试：核心流程全覆盖
- E2E 测试：Playwright 自动化
- 视觉回归测试：像素对比

### 5.2 性能要求
| 指标 | 目标 |
|------|------|
| 首次加载 | < 3s |
| 页面切换 | < 500ms |
| 发布响应 | < 2s |

### 5.3 安全要求
- 账号密码 AES-256-GCM 加密
- 敏感信息不日志输出
- HTTPS 传输

## 6. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 技术栈差异 | 代码不能直接复用 | 参考设计重写 |
| 功能复杂度 | 开发周期长 | 分阶段实施 |
| 平台兼容性 | 不同平台行为差异 | 充分测试 |
| 第三方依赖 | API 变更影响 | 版本锁定 |

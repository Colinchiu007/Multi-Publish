## 1. 产品概述

### 1.1 产品定位
蚁小二是一款多平台内容发布管理工具，帮助用户统一管理多个社交媒体账号，一键发布内容到多个平台。

### 1.2 目标用户
- 自媒体运营者
- 企业新媒体部门
- 内容创作者
- KOL/网红

### 1.3 核心价值
- 节省多平台切换时间
- 统一管理账号和内容
- 数据统计分析
- 团队协作

## 2. 功能模块分析

### 2.1 首页/工作台 (Home)

#### 功能描述
- 展示关键数据概览（发布数量、粉丝增长、数据趋势）
- 快捷操作入口
- 最近操作记录
- 账号状态总览

#### UI 元素
| 元素 | 说明 |
|------|------|
| 顶部导航栏 | 用户信息、通知、设置 |
| 数据卡片 | 展示关键指标 |
| 快捷操作区 | 快速发布、添加账号等 |
| 最近动态 | 时间线形式展示 |

#### 对应 Multi-Publish 现状
- **已有**: 基础首页展示
- **缺失**: 详细数据统计、快捷操作卡片
- **复用优先级**: P1

---

### 2.2 内容发布 (Publish) ⭐⭐⭐ 核心功能

#### 功能描述
支持多平台内容发布，包括图文和视频。

#### 2.2.1 内容编辑器
| 功能 | 描述 |
|------|------|
| 富文本编辑 | 支持文字、图片、视频混合编辑 |
| Markdown 支持 | 代码块、列表等格式 |
| 图片上传 | 本地上传、素材库选择 |
| 视频上传 | 支持主流格式 |
| 封面选择 | 自动提取/手动上传 |
| 话题标签 | #话题 自动补全 |
| @提及 | @用户 自动补全 |

#### 2.2.2 平台选择
| 功能 | 描述 |
|------|------|
| 多平台选择 | 微博、知乎、头条、抖音、B站等 |
| 平台分组 | 按类型分组（图文/视频/社交） |
| 差异化设置 | 每个平台可单独设置内容 |
| 平台兼容性 | 自动适配各平台格式限制 |

#### 2.2.3 发布选项
| 选项 | 描述 |
|------|------|
| 立即发布 | 审核后立即发布 |
| 定时发布 | 设置具体时间 |
| 草稿保存 | 保存到草稿箱 |
| 定时循环 | 周期性发布 |

#### UI 流程
```
内容编辑 → 平台选择 → 发布选项 → 确认发布 → 状态跟踪
```

#### 对应 Multi-Publish 现状
- **已有**: 基础发布功能、RPA 发布引擎
- **缺失**: 富文本编辑器、差异化内容设置、定时发布
- **复用优先级**: P0（必须）

| 指标 | 要求 |
|------|------|
| 启动时间 | < 3秒 |
| 页面切换 | < 500ms |

### 3.2 兼容性
| 平台 | 要求 |
|------|------|
| Windows | Windows 10+ |

- 敏感信息不日志输出

## 4. 功能优先级

### P0 - 核心功能（必须实现）
1. 账号登录与状态持久化
2. 账号管理（添加/删除/状态检测）
3. 内容发布（图文/视频）
4. 平台选择与发布

### P1 - 重要功能（应该实现）
5. 扫码登录
6. 草稿箱
7. 定时发布
8. 评论管理
9. 素材库
10. 基础设置

### P2 - 增值功能（可以实现）
11. 数据统计
12. 团队管理
13. AI 助手
14. 工具箱
15. 高级数据分析

## 5. 技术架构建议

### 5.1 前端架构
- **框架**: Vue 3 + Vite
- **状态管理**: Pinia
- **UI 组件**: Element Plus / Ant Design Vue
- **富文本编辑器**: TipTap / Quill

### 5.2 后端架构
- **主进程**: Electron
- **数据存储**: SQLite (sql.js) + localStorage
- **发布引擎**: RPA + API 混合模式

### 5.3 复用策略
| 模块 | 复用方式 |
|------|---------|
| 账号管理 | 参考蚁小二 UI，复用 Multi-Publish credential-store.js |
| 内容发布 | 集成蚁小二编辑器设计，扩展现有发布模块 |
| 素材库 | 新开发，参考蚁小二设计 |
| 评论管理 | 扩展现有评论模块 |
| 设置 | 扩展现有设置模块 |

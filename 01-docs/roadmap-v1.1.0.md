# v1.1.0 路线图 — 从研发到运营

> 目标：稳定产品基础 → 获取首批 200 个活跃用户 → 验证付费模型

---

## 里程碑

| 阶段 | 时间 | 目标 |
|------|------|------|
| **M1: 产品稳定** | 当前 → W2 | 修复所有已知 Bug，确保 10 平台可用 |
| **M2: 运营启动** | W2 → W4 | 首批 1,000 下载，200 活跃用户 |
| **M3: 付费闭环** | W4 → W6 | Pro 版上线，> 8% 转化率 |
| **M4: 迭代增长** | W6+ | 用户反馈驱动迭代，拓展功能 |

---

## v1.1.0 功能清单

### P0（产品稳定 — 必须）
- [ ] **自动更新修复验证** — v1.0.7 已含修复，端到端测试
- [ ] **Python 后端稳定性** — 进程守护，崩溃自动重启
- [ ] **Python Core 模块合并** — QueryWorker/TaskScheduler/Progress/Downloader/DataSync
- [ ] **新平台端到端测试** — 10 平台账号添加 + 发布验证

### P1（用户体验 — 重要）
- [ ] **操作引导** — 首次启动流程引导（step-by-step）
- [ ] **发布失败告警** — 通知栏 + 声音提示
- [ ] **英文版界面** — i18n，覆盖海外用户
- [ ] **离线模式** — 网络中断不影响已登录的 Playwright 发布

### P2（运营支撑）
- [ ] **支付接入** — 支付宝/微信支付 SDK 集成
- [ ] **付费兑换码系统** — 生成 + 验证
- [ ] **使用量统计** — 埋点 + 后台看板
- [ ] **登录试用倒计时** — Pro 7 天限时体验

### P3（增长功能）
- [ ] **内容模板** — 预设模板库（快速发布常用格式）
- [ ] **AI 辅助写作** — 标题生成/正文摘要（集成 LLM API）
- [ ] **发布日历** — 可视化内容排期
- [ ] **数据看板增强** — 阅读量/互动数据回流

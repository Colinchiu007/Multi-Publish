# last30days × Multi-Publish 集成分析

> **结论先行：可以，但不是直接搬。last30days 的核心价值（跨平台社交搜索 + 真实互动评分）可以作为 Multi-Publish 的「内容研究」模块，补上当前缺失的「发布前调研」和「发布后影响追踪」能力。**
>
> 推荐：**「内容情报面板」方案**，作为 Multi-Publish 的 F11 新功能，嵌入现有采集流程。

---

## 1. Multi-Publish 当前缺失的能力

Multi-Publish 当前的内容工作流是：

```
采集 → 编辑 → 发布 → 监控（仅审核状态）
```

缺少两个关键环节：

| 缺失环节 | 问题 | last30days 能补什么 |
|---------|------|-------------------|
| **发布前调研** | 不知道目标平台当前在 hot 什么、用户的同类内容表现如何 | 跨平台搜索 + engagement 评分，帮创作者选角度/标题/时机 |
| **发布后影响** | 仅追踪审核状态，不追踪内容发布后的社交反响 | 发布后持续追踪各平台的讨论/提及/二次传播 |

---

## 2. 三个可行的集成方案

### 方案 A：内容情报面板（推荐 MVP）— 3-5 人日

在 Multi-Publish 中新增一个「情报」页签，作为 F5 内容采集的扩展。

**工作流：**

```
[情报面板]
  ├─ 输入主题关键词
  ├─ last30days 引擎搜索 17 平台
  ├─ 展示热度趋势 + 热门内容 + 高互动标题
  └─ 用户选中某条结果 → 自动填入编辑器作为灵感/参考
      └─ 或基于热门标题生成相似标题（AI rewrite）
```

**用户场景：**
1. 准备发一篇小红书/抖音 → 先在情报面板搜"短视频 2026 趋势"
2. 看到哪些标题互动高、哪些角度在同质化竞争
3. 调整选题/标题后再发布

**架构落地：**

```
apps/desktop/electron/
  ├── content-intelligence.js        ← NEW: 情报引擎（封装 last30days 逻辑）
  ├── url-collector.js               ← 已有：URL 采集（可引出情报结果）
  └── content-aggregator-bridge.js   ← 已有：可兼做情报数据通道

apps/desktop/src/
  ├── views/
  │   ├── Publish.vue                ← 已有
  │   └── Intelligence.vue           ← NEW: 情报面板页面
  └── components/
      └── IntelligencePanel.vue      ← NEW: 情报组件（可嵌入编辑侧栏）
```

**数据流：**

```
Intelligence.vue
  → IPC: content-intelligence:search
  → content-intelligence.js
     → 跨源 HTTP 请求 (Reddit/HN/GitHub 等免费源)
     → engagement 归一化评分 (log10 公式)
     → 按平台分组 + 按热度排序
  → 返回前端展示
  → 用户点击某条 → 填充编辑器
```

**免费数据源（零成本起步）：**
- Reddit (公开 JSON)
- Hacker News (Algolia API)
- GitHub API
- YouTube (yt-dlp 本地工具，已安装则可用)

**如果要覆盖中国平台（小红书/抖音/B站）：**
- 需要额外的 spider 或第三方 API
- 初期可以用 Reddit/HN/YouTube 三大国际源覆盖通用趋势
- 中国平台可作为 P2 后补

---

### 方案 B：发布标题优化助手 — 2-3 人日

在现有编辑器中嵌入「助手侧栏」，当用户输入标题时实时搜索同类内容。

**工作流：**

```
[编辑器中输入标题]
  → 防抖 2 秒后自动触发 last30days 搜索
  → 侧栏展示：
     - 同类标题在各平台的互动数据（点赞/评论/转发）
     - 建议替换关键词（基于高互动标题的 TF-IDF 分析）
     - 热度趋势（上升/下降/平稳）
  → 用户可一键采纳建议标题
```

**嵌入位置：**

```
Publish.vue (编辑器页面)
  └── ArticleEditor.vue (已有)
      └── TitleAssistantPanel.vue  ← NEW: 标题建议侧栏
```

**优势：** 侵入最小，不破坏现有发布流程。
**劣势：** 只影响标题，不影响选题层决策，价值相对有限。

---

### 方案 C：发布后影响力追踪 — 4-6 人日

在现有 F6 发布历史中新增「影响力」维度的追踪。

**工作流：**

```
[发布完成]
  → 发布后 T+1h / T+24h / T+72h 各触发一次
  → last30days 搜索发布内容的标题+关键词
  → 统计各平台的讨论量、提及量、情感倾向
  → 展示在统计看板的时间轴上
```

**架构落地：**

```
electron/
  ├── publish-monitor.js            ← 已有：审核状态监控
  └── publish-impact-tracker.js     ← NEW: 影响力追踪

src/views/
  └── Dashboard.vue                 ← 已有统计看板
      └── 新增影响力图表组件
```

**优势：** 补全了内容生产的最后闭环（发出去了→效果如何）。
**劣势：** 依赖较多 API 调用，且准确定位内容在各平台的二次讨论需要较好的搜索精确度。

## 3. 三个方案对比

| 维度 | A: 情报面板 | B: 标题助手 | C: 影响力追踪 |
|------|------------|------------|-------------|
| **用户价值** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| **开发成本** | 3-5 人日 | 2-3 人日 | 4-6 人日 |
| **侵入性** | 新增页面，零侵入 | 嵌入侧栏，低侵入 | 扩展现有模块 |
| **依赖** | 免费 API 即可起步 | 标题防抖搜索 | 定时任务 + 搜索 |
| **可验证性** | 高（搜索→看结果） | 高（标题→看建议） | 中（准确定位难） |
| **风险** | 低 | 低 | 中（噪音可能大） |

**推荐：先做 A（情报面板）**，5 天完成核心价值。B 和 C 可在 A 的基础上复用搜索引擎，渐进添加。

---

## 4. 与 TrendScope 定位差异

TrendScope 是**热榜发现**（被动看平台热门榜单），last30days 是**主题搜索**（主动搜某个关键词在各平台的讨论）。两者在 Multi-Publish 中互补：

| | TrendScope | last30days |
|---|-----------|------------|
| 模式 | 平台热榜 → 发现趋势 | 关键词搜索 → 获取讨论 |
| 适合 | 找"今天什么🔥" | 找"关于 X 大家都在说什么" |
| Multi-Publish 集成 | 间接（数据来自上游管道） | 直接（嵌入情报面板） |

Multi-Publish 更适合 last30days，因为发布者有明确的主题意图（我要发一篇关于 X 的内容），需要的是主题深度的社交情报，而非泛热榜。

## 5. 实施建议

**Phase 1（3-5 人日）— 情报面板 MVP：**

1. 创建 `content-intelligence.js`，封装 last30days 的核心逻辑
   - 支持免费源（Reddit + HN + GitHub + YouTube）
   - engagement 归一化评分
   - 按平台分组展示
2. 创建 `Intelligence.vue` 页面 + `IntelligencePanel.vue` 组件
3. IPC 通道注册 `content-intelligence:search`
4. 集成点：情报结果选中后 → 填充到编辑器

**Phase 2（2-3 人日）— 嵌入编辑流程：**

1. 在编辑器中添加「情报侧栏」按钮
2. 输入标题自动触发关联搜索
3. 标题建议 + 标签建议

**Phase 3（4-6 人日）— 影响力追踪（可选）：**

1. 发布后定时任务
2. 跨平台提及检测
3. 看板集成

---

## 6. 风险

| 风险 | 等级 | 缓解 |
|------|------|------|
| 免费源不稳定 | 🟡 | 每个平台有 fallback，源故障不影响主发布流程 |
| 搜索结果噪音 | 🟡 | last30days 的 engagement 评分 + 聚类算法可缓解 |
| 中国平台数据缺失 | 🟡 | 初期声明仅国际源，中国平台 P2 补充 |
| 用户学习成本 | 🟢 | 情报面板是独立页签，不影响现有发布流程 |

---

## 7. 结论

> **last30days 在 Multi-Publish 中的价值高于在 TrendScope 中的价值。**
>
> TrendScope 已有自己的热榜发现管道，last30days 是同类功能做不同的事（热榜 vs 搜索）。但 Multi-Publish 完全没有发布前调研能力，last30days 正好补这个空白。
>
> **推荐路线：直接做方案 A（情报面板），3-5 人日交付，零成本起步。**

## 二、F1：格式适配器

### 2.1 需求

每条内容从编辑器到各平台，需要自动转换格式：

```
编辑器(HTML/Markdown)
        │
        ├── 微信公众号 → 特殊微信图文HTML（带样式映射）
        ├── 知乎 → Markdown 增强（LaTeX/代码块兼容）
        ├── 微博 → 纯文本 truncate（140字/2000字限制）
        ├── 抖音 → 纯文本 + 话题标签
        ├── 小红书 → 纯文本 + Emoji + 话题
        ├── 视频号 → 纯文本 + 话题
        ├── 快手 → 纯文本
        ├── 今日头条 → Markdown 增强
        ├── YouTube → 纯文本 + 话题标签
        ├── TikTok → 纯文本 + 话题
        └── B站 → Markdown 增强（BBCode 标签转换）
```

### 2.2 验收标准

- [ ] 编辑器内容 → 各平台格式正确，无乱码/丢失
- [ ] 标题长度自动截断（各平台上限不同）
- [ ] 正文图片 URL 全部保留
- [ ] 话题/标签格式正确
- [ ] 有回退机制（目标格式不支持 → 降级为纯文本）

## 三、F2：封面图自动处理

### 3.1 需求

各平台封面图尺寸/格式要求不一，自动处理：

| 平台 | 推荐尺寸 | 格式 | 最大大小 |
|------|---------|------|---------|
| 微信公众号 | 900×500 | JPEG | 10MB |
| 知乎 | 1280×720 | JPEG/PNG | 5MB |
| 微博 | 980×550 | JPEG/PNG | 20MB |
| 抖音 | 1080×1440 | JPEG/PNG | 20MB |
| 小红书 | 1080×1080 | JPEG/PNG | 10MB |
| 头条 | 1200×600 | JPEG | 5MB |
| B站 | 1146×717 | JPEG/PNG | 5MB |
| YouTube | 1280×720 | JPEG/PNG | 2MB |

### 3.2 验收标准

- [ ] 自动裁剪到目标尺寸（中心裁剪）
- [ ] 自动压缩到大小限制内
- [ ] 格式转换（PNG → JPEG 等）
- [ ] 原图保留，不破坏原文件
- [ ] 处理失败时使用原图发布（不阻塞）

## 四、F3：更多平台（百家号/搜狐号/企鹅号）

### 4.1 需求

新增 3 个中文内容平台：

| 平台 | 模式 | 发布类型 | 优先级 |
|------|------|---------|:------:|
| **百家号 (baijiahao)** | RPA | 图文/视频 | P1 |
| **搜狐号 (souhu)** | RPA | 图文 | P1 |
| **企鹅号 (qiehao)** | RPA | 图文/视频 | P1 |

**特点**：
- 三个平台均为纯 RPA 模式（无公开 API）
- 需要 Playwright 浏览器自动化
- 登录方式：Cookie（扫码/密码登录后保存）

### 4.2 验收标准

- [ ] 3 个平台 RPA 发布器语法完整
- [ ] 注册到 registry，前端可见
- [ ] 发布流程：登录 → 填写 → 发布 → 验证结果
- [ ] 标签/话题格式适配（依赖 F1）
- [ ] 封面图尺寸适配（依赖 F2）

## 三、F1：平台 URL 配置化

### 需求

将所有平台信息从散落的代码中提取到统一配置文件 `config/platforms.yaml`：

```yaml
platforms:
  weixin:
    id: 1
    name: 微信公众号
    type: article
    publish_url: ""
    data_url: ""
    comment_url: ""
    income_url: ""
    cover_size: "900x500"
    max_title: 64
    categories: "weixin_categories.yaml"
```

### 受益方

- **发布器**：从配置读取发布 URL
- **数据同步**：从配置读取数据页 URL
- **评论管理**：从配置读取评论页 URL
- **F2 封面图**：从配置读取 `cover_size`
- **F1 格式适配**：从配置读取 `max_title`

### 验收

- [ ] `config/platforms.yaml` 覆盖所有 12 个当前平台
- [ ] 从配置读取平台信息替代硬编码
- [ ] 加载失败有回退机制
- [ ] 新增平台只需改 YAML

## 四、F2：敏感词预检

### 需求

发布前检查内容中是否包含敏感词，避免发出后被删除。

- 融媒宝词库 812KB（`sensitivelibrary.txt`）
- 检查范围：标题 + 正文
- 触发时机：点击发布前
- 反馈：提示具体敏感词位置

### 验收

- [ ] 加载敏感词库文件
- [ ] 标题+正文扫描
- [ ] 命中敏感词 → 提示用户修改
- [ ] 无命中 → 正常发布
- [ ] 语料库不存在 → 跳过（不阻塞）

---

## 五、F3：数据同步系统

### 需求

从各平台拉取真实数据：阅读量、粉丝数、收益、评论数。

- 参考融媒宝 `data/media_url_list.txt` 中的 data_url
- 项目003 已有 `AggregatorBridge` 和 `DataSyncService` 骨架
- 输出到 Dashboard 前端展示

### 验收

- [ ] 至少 5 个主流平台的数据拉取配置
- [ ] 数据展示在 Dashboard
- [ ] 定时刷新（每小时/每天）
- [ ] 拉取失败跳过（不阻塞）

---

## 六、F4：评论统一管理

### 需求

统一查看和回复各平台评论。

- 参考融媒宝评论 URL 配置
- 用 WebContentsView 打开各平台评论管理页
- 或通过 API 拉取评论列表

### 验收

- [ ] 至少 3 个平台的评论页面导航
- [ ] 统一收件箱视图
